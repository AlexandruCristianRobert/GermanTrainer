# GermanTrainer Repository: B2–C1 Module Expansion Plan

## TL;DR
- The repo is publicly accessible: it's a personal-use Vue 3 + TypeScript + Vite SPA with Dexie/IndexedDB storage and direct Anthropic API calls, currently containing only two modules — a noun gender/translation quiz and an LLM-generated adjective fill-in-the-blank quiz — neither of which targets B2–C1 specifically.
- The biggest learning gaps versus what serious B2–C1 platforms (Goethe-Institut, DW Deutsch lernen, Lingoda, Easy German/Seedlang, DWDS, telc) actually drill are advanced grammar transformations (Konjunktiv I/II, Passiv alternatives, Nominalisierung), exam-format productive tasks (Goethe C1 Schreiben, telc Sprachbausteine), idiom/collocation depth, and native-speed listening with shadowing/dictation.
- I recommend ten concrete, non-overlapping new modules below — each maps cleanly onto your existing stack (Vue 3 component + Dexie store + Claude prompt template) and onto an inspiration platform, with most leaning on the Anthropic API you already wire in for generation and grading.

## Key Findings

### 1. What the repo already contains
Direct inspection of `https://github.com/AlexandruCristianRobert/GermanTrainer` (README, file tree, package.json) shows:

- **Tech stack** — Vue 3, Vite, TypeScript, Naive UI, Vue Router, Dexie (IndexedDB), `@anthropic-ai/sdk`. All persistence is local IndexedDB; the Anthropic API is called directly from the browser. Default model is `claude-sonnet-4-6` with `claude-haiku-4-5-20251001` as a cheaper fallback.
- **Project layout** — `src/`, `tests/`, `docs/superpowers/specs/2026-05-04-german-trainer-design.md`, `docs/superpowers/plans/2026-05-04-german-trainer.md`, plus `src/data/*.seed.json` seed lists managed through "Manage nouns" / "Manage adjectives" UI.
- **Existing learning modules** — only two:
  1. **Nouns** — quizzes the gender (`der`/`die`/`das`) or English translation of German nouns from a seed list.
  2. **Adjectives** — fill-in-the-blank quiz where Claude generates sentences on demand around target adjectives.
- **Deployment posture** — explicitly personal/local use only ("Do not deploy this as a public site — there is no backend proxy"). The API key sits in IndexedDB.

This is essentially an A2–B1 vocabulary/declension trainer at present. None of the four CEFR skills beyond word-level recognition (no listening, no extended writing, no exam tasks, no grammar transformations, no collocation work, no idioms, no register practice) are covered yet.

### 2. What B2–C1 platforms actually drill
Synthesis from the platforms researched:

- **Goethe-Institut (Goethe-Zertifikat C1, modular since 1 Jan 2024)** — Reading (65 min, four task types totalling 30 items, including a summary-with-gap-fill and a multiple-choice cloze), Listening (40 min, podcast + interview + discussion + lecture), Writing (75 min for a ~230-word forum post + ~120-word semi-formal email), Speaking (paired, lecture + discussion). The official Modellsatz (Feb 2023) states verbatim: *"In der Prüfung können maximal 100 Punkte pro Modul erreicht werden. Die Bestehensgrenze liegt bei 60 Punkten, also 60 Prozent."* The free Goethe "Deutsch für dich" portal currently advertises *"There are over 200 exercises"* across A1–C2 (the older community/social feature has been discontinued; only the self-study exercises remain).
- **Deutsche Welle Deutsch lernen** — *Nicos Weg* (A1–B1, video + interactive exercises); for B2–C1 the flagships are **Langsam gesprochene Nachrichten** (daily DW news read slowly, with full text and downloadable MP3), **Top-Thema mit Vokabeln** (twice-weekly news reports plus vocabulary annotations and comprehension questions), **Deutsch im Fokus** (society reportages with explanations and multiple-choice comprehension), and **Das sagt man so!** (≈2-minute idiom explainer podcast).
- **Lingoda B2/C1** — live small-group classes; B2 syllabus explicitly drills Konjunktiv I + modal-passive ("Modalverben mit Passiv und Konjunktiv I"), short literary genres, public-administration register; C1 emphasises formal/administrative register, implicit meaning, and complex texts.
- **Seedlang/Easy German** — video flashcards using vox-pop interviews; built-in pronunciation comparison (record + compare to native), interactive video stories, gender/plural/conjugation trainers, "click-any-word for grammar" reveal. Easy German Membership adds a podcast with interactive transcripts + a "vocabulary helper" overlay and PDF worksheets.
- **LingQ** — import any German YouTube/Netflix/podcast/article, click unknown words to mark and translate, SRS review, automatic progress tracking by total known-word count.
- **Anki/Memrise** — pure SRS flashcards, user-generated decks; relevant as a model for scheduling logic.
- **Duolingo** — gamified tree; no real C1 content for German.
- **Slow German with Annik Rubens / News in Slow German** — slow-paced authentic-style audio with transcripts (B1–C1 tracks).
- **GermanPod101, Learn German with Anja** — video lessons with topic-based vocab and dialogue + transcript, weighted to A2–B2.
- **Verbformen.de / Cactus2000 / scholingua / german.net** — conjugation trainers; verbformen.de also offers full declension tables for nouns/adjectives and worksheets.
- **DWDS (Berlin-Brandenburg Academy)** — definitions + etymology + frequency timelines + collocations. The Wortprofil page (dwds.de/d/wortprofil, updated 15 Jan 2026) states verbatim: *"Das Wortprofil bietet für 400 000 Lemmaformen (Abfragewörter) insgesamt 22,5 Millionen verschiedene Kookkurrenzen (inklusive symmetrischer Relationen) mit knapp 1 Milliarde Korpustreffern."* DWDS exposes a public API for `frequency` and `snippet` lookups.
- **AI tutors (DeutschExam.ai, Viobean, Talkpal)** — instant grammar/coherence/vocabulary feedback against Goethe/telc rubrics; AI conversation partners that evaluate pronunciation, grammar, fluency, and task completion per turn.
- **Exam-format demands you must drill at B2–C1** — Konjunktiv I for indirect speech (essential in news/academic writing), Passiv plus alternatives (Vorgangs-/Zustandspassiv, `sich lassen`, `sein + zu`, `-bar`-Adjektive), Nominalisierung ↔ Verbalisierung, complex Konnektoren (kausal, konzessiv, konsekutiv, adversativ, modal — `infolgedessen`, `mithin`, `wenngleich`…), Partizipialkonstruktionen as compressed relative clauses, register switching (neutral ↔ formal ↔ academic), idioms (Redewendungen), and C-Test/cloze under time pressure.

## Details — 10 Concrete Module Suggestions

Each module is sized to drop into the existing repo as **one Vue 3 route** (e.g. `src/modules/<name>/`) with its own Dexie table and an optional Claude prompt template. None overlaps with the existing Nouns/Adjectives modules.

### 1. Goethe C1 Schreiben Simulator (Forumsbeitrag + formelle E-Mail)
**Inspiration:** Goethe-Zertifikat C1 modular Schreiben task — 75 minutes total, ~230-word forum post + ~120-word semi-formal email. The official Modellsatz specifies *"Diskussionsbeitrag in einem Online-Forum … Freier Text (ca. 230 Wörter). Umsetzung von vier verschiedenen Sprachfunktionen"* for task 1 and *"(halb-)formelle Mitteilung verfassen … E-Mail … Freier Text (ca. 120 Wörter)"* for task 2. Pass mark 60/100 per module (*"Die Bestehensgrenze liegt bei 60 Punkten, also 60 Prozent."*). **Data needed:** a seeded JSON of ~30 prompt pairs (one social-issue forum theme + one related formal-message scenario) plus the four Sprachfunktionen the rubric checks for each task. **UI/UX:** a 75-minute countdown, dual-pane editor with live word counter and target band coloring (red <90% of target, green within ±15%), and a "Submit for grading" button that POSTs the text to Claude with a Goethe-rubric prompt (Erfüllung, Kohärenz, Wortschatz, Strukturen, Korrektheit) and returns a per-criterion score plus inline diff. **Learning value:** the single highest-leverage C1 productive task; most candidates fail on word-count discipline, register, and Sprachfunktion coverage rather than grammar.

### 2. telc C1 Sprachbausteine Trainer (22-gap × 4-option cloze under timer)
**Inspiration:** telc Deutsch C1 Sprachbausteine. The official telc Übungstest specification table states verbatim: *"2 Sprachbausteine · Grammatik und Lexik · 22 4er-Mehrfachwahlaufgaben · 22 [Punkte] · 22 [Minuten]"* — so 22 four-option multiple-choice gaps in a continuous text, 22 minutes, 22 points. (telc C1 Hochschule uses 16 gaps in a wissenschaftssprachlich text and is the variant for university applicants.) **Data needed:** a Dexie table of generated cloze passages tagged by topic (Wirtschaft, Wissenschaft, Politik…) and by what each gap tests (Konnektor, Präposition, Verb-Rektion, fester Ausdruck, Modalpartikel). Use Claude to generate passages on demand from a topic seed, then have a second Claude pass solve them to validate that exactly one option is unambiguously correct. **UI/UX:** 22-minute countdown, click-to-select options inline in the passage, per-gap explanation revealed after submit. **Learning value:** Sprachbausteine cleanly forces lexicogrammar — exactly the C1 register/collocation gaps that text-generation alone hides.

### 3. Konjunktiv I "Indirekte Rede" News Rewriter
**Inspiration:** DW Langsam gesprochene Nachrichten and Tagesschau formal register, where Konjunktiv I dominates. **Data needed:** short authentic-style direct-speech quotes ("Der Minister sagte: 'Wir werden …'"). Use Claude to generate quote/news-sentence pairs at B2/C1 difficulty; store reference Konjunktiv I and fallback-to-II forms. **UI/UX:** show the direct quote, ask the user to rewrite as `Der Minister sagte, … + Konjunktiv I`, score against the reference, and explain when Konjunktiv II is mandated (form coincides with indicative — e.g. `sie kommen` → `sie kämen`). **Learning value:** Konjunktiv I is the most under-drilled but most distinctive register marker at C1 — used heavily in TestDaF and Goethe written tasks.

### 4. Passiv Transformation & Alternatives Drill
**Inspiration:** Lingoda B2 chapter on "Modalverben + Passiv + Konjunktiv I" plus the C1 grammar guides emphasising Vorgangs-/Zustandspassiv and passive substitutes (`sich lassen + Infinitiv`, `sein + zu + Infinitiv`, `-bar`/`-lich` adjectives, `man`-Konstruktion). **Data needed:** seeded active sentences tagged with which transformations are legal (intransitive verbs allow none; transitives allow all). **UI/UX:** present an active sentence, show six tabs (Vorgangspassiv, Zustandspassiv, sich-lassen, sein+zu, -bar, man-Konstruktion), user types each legal form; the app marks impossible tabs disabled and grades the rest with Claude as fallback grader. **Learning value:** the active↔passive↔alternative switch is the bread and butter of academic register and is rarely drilled systematically.

### 5. Nominalisierung ↔ Verbalisierung Switcher
**Inspiration:** Lingoda C1 grammar list and the Mittelpunkt textbook approach — academic German runs on noun-heavy style ("Durch den Vergleich werden Unterschiede sichtbar." vs. "Wenn man vergleicht, erkennt man Unterschiede."). **Data needed:** sentence pairs in both styles, tagged with the trigger pattern (Präpositionalphrase ↔ Nebensatz, deverbal noun ↔ finite verb, `bei`/`durch`/`trotz`/`statt` + Nominalisierung). **UI/UX:** two-direction drill — show one form, ask for the other; reveal the morphology rule (e.g. `vergleichen → der Vergleich`, `erkennen → die Erkenntnis`) on review. Claude generates new pairs from a topic. **Learning value:** the #1 stylistic marker separating B2 from C1 writing in TestDaF/Goethe rubrics.

### 6. Konnektoren & Register Polyphony Trainer
**Inspiration:** Goethe/telc C1 register expectations and the atozgrammatik C1 connector list (kausal, konsekutiv, konzessiv, adversativ, modal, temporal, konditional). **Data needed:** a sentence with one connector slot and three equivalent connectors at different registers (e.g. neutral `weil` / formal `da` / academic `zumal`/`infolgedessen`; or concessive `obwohl` / `wenngleich` / `ungeachtet dessen`). **UI/UX:** show a sentence + target register tag ("akademisch", "formell", "neutral", "umgangssprachlich"); user picks the right connector and adjusts word order (Hauptsatz vs. Nebensatz). Streak-track which connector families the user keeps missing. **Learning value:** raw vocabulary knowledge of `dennoch` vs. `obgleich` is useless unless paired with register tagging — exactly what C1 rubrics reward.

### 7. Redewendungen Trainer ("Das sagt man so!" style)
**Inspiration:** DW Deutsch lernen *Das sagt man so!* (≈2-min idiom explainer episodes), Goethe-Institut's published B1–C2 idiom list, the *Sprichwort-Spaß* card-game pedagogy (draw / fill in the blank / define / guess from picture). **Data needed:** a Dexie table of ~300–500 high-frequency idioms with: literal translation, figurative meaning, register tag (umgangssprachlich/neutral/gehoben), one authentic example sentence, optional emoji/image. Source candidates: German Wiktionary "Redewendung" category (CC BY-SA, bulk-extractable via `wiktionary-de-parser`) and curated lists; do not scrape Redensarten-Index.de (no API, owner-supported). **UI/UX:** four drill modes — multiple-choice meaning, fill-in-the-blank in an example sentence, reverse (meaning → idiom), and "guess from emoji clue". Claude generates fresh example sentences on demand. **Learning value:** idiom recognition is the visible difference between B2 and C1 listening/reading scores; almost no app drills it systematically.

### 8. Shadowing & Dictate Listening (DW Langsam → normal speed ladder)
**Inspiration:** Easy German's authentic-speech philosophy + News in Slow German's pace ladder + DW Langsam gesprochene Nachrichten (which publishes daily slow news with a full transcript). **Data needed:** a curated playlist of 30–60-second audio clips at three speeds — slow (DW LGN-style), normal news pace, and casual conversational (Easy German vox-pop). **Do not bundle DW audio**; deep-link to the DW episode URL and store only your own metadata + the transcript snippet (DW content © Deutsche Welle, link-out is fine). **UI/UX:** two modes — (a) **Shadowing**: play sentence, record user, use the browser MediaRecorder API + a speech-to-text service to compare token-by-token, highlight mismatches; (b) **Dictation**: play 5–15-second chunk, user types, app diff-grades against the reference transcript. **Learning value:** the single biggest C1 listening complaint is "I can read it but can't hear it at speed" — shadowing/dictation directly bridges that.

### 9. Collocation / Wortprofil Trainer
**Inspiration:** DWDS Wortprofil (the Wortprofil 2026v1 build covers 400 000 lemma forms with 22.5 million distinct co-occurrences from ~13 million texts / 286 million sentences / 7.5 billion text words) and the Wortschatz Leipzig API. **Data needed:** for each high-frequency C1 noun/verb, the top 10 collocates by syntactic relation (e.g. for `Entscheidung`: `eine Entscheidung treffen / fällen / revidieren / anfechten`; modifiers: `weitreichend, einstimmig, folgenschwer`). Pull at build time from the DWDS public API (`/api/frequency`, `/api/wb/snippet`) and the open-source `zentrum-lexikographie/wordprofile` repo (GPL-3.0), or from Leipzig Corpora Collection downloads (`deu_news_2024_*`). **UI/UX:** "which verb collocates?" multiple choice (`* eine Entscheidung` → treffen ✓ / machen ✗ / nehmen ✗); plus reverse drill — given a noun, type as many partner verbs as possible in 60 s. **Learning value:** native-like collocation is the strongest sign of C1+; word-level vocab without collocations is what makes intermediate German sound translated.

### 10. AI Writing Tutor with Goethe/telc Rubric Grading
**Inspiration:** DeutschExam.ai, Viobean, Cambridge Writing Checker (analogue), and the per-criterion rubric grading available via prompting Claude. **Data needed:** a Dexie table of prompts (Forumsbeitrag, formelle E-Mail, argumentativer Aufsatz, Grafik-Beschreibung, Zusammenfassung, Stellungnahme) each with the official Goethe or telc criteria array baked into a Claude prompt template. **UI/UX:** a free-form rich-text editor, a "Grade me" button that returns (a) overall band estimate (B2 / C1- / C1 / C1+), (b) per-criterion scores (Erfüllung, Kohärenz, Wortschatz, Strukturen, Korrektheit) with verbatim citations from the text, (c) an inline diff with suggested rewrites, and (d) an "Improve this paragraph using more C1-style nominalization/connectors" button that demonstrates an upgraded version side-by-side. Cache graded essays in IndexedDB so the user can compare drafts over time. **Learning value:** the bottleneck for self-study B2→C1 writing has always been "nobody corrects me" — your existing Anthropic-SDK wiring makes this the lowest-effort, highest-value module to ship next.

## Recommendations (priorities, staged)

**Ship next (Sprint 1, ~1–2 weeks each):**
1. **Module 10 (AI Writing Tutor)** — almost zero new infrastructure; reuses the API key, Claude SDK, and Dexie pattern you already have. This alone closes the biggest B2→C1 gap.
2. **Module 3 (Konjunktiv I)** and **Module 4 (Passiv transformations)** — small seed data, mostly Claude-generated, extends your current "fill-in-the-blank" muscle.

**Sprint 2 (~2–4 weeks):**
3. **Module 1 (Goethe C1 Schreiben Simulator)** — natural extension of Module 10, adds the timed/word-count exam format.
4. **Module 2 (telc Sprachbausteine)** — high reuse of generation+validation pipeline from Module 1.
5. **Module 7 (Redewendungen)** — needs a one-time seed harvest from de.wiktionary (CC BY-SA) plus Goethe's published idiom list.

**Sprint 3 (~4+ weeks):**
6. **Modules 5, 6, 9 (Nominalisierung, Konnektoren register, Collocations)** — these need quality seed data: pull from Leipzig Wortschatz downloads (`https://downloads.wortschatz-leipzig.de/corpora/deu_news_2024_*.tar.gz`) and DWDS Wortprofil (run the GPL-3.0 backend `zentrum-lexikographie/wordprofile` locally to build your own collocation dump).
7. **Module 8 (Shadowing/Dictation)** — last, because audio recording, browser permissions, and speech-to-text add the most infrastructure complexity.

**Benchmarks that should change priorities:**
- If your goal is a specific Goethe/telc/TestDaF date within 3 months → bump Modules 1, 2, 10 to top regardless.
- If you want to plateau-bust without an exam → Modules 5, 6, 9 (style/collocations) deliver the most "sounding native" mileage.
- If shadowing infrastructure proves too heavy (browser audio API friction) → swap Module 8 for an LLM-based "transcript-without-audio" pseudo-shadowing that scores rhythm-marked sentences.

## Caveats

- **Repo visibility** — the repository was publicly readable when fetched; the task description's "private repo" note was inaccurate at access time. Findings are based on the live README, file tree, and the explicit `package.json` and `docs/superpowers/specs/2026-05-04-german-trainer-design.md` references in the README.
- **Personal-use posture** — the README is emphatic that this is a single-user app with the Anthropic key in IndexedDB; every suggestion above respects that (browser-only, no backend proxy, no shared deployment). If you ever publish it, all ten modules would need a backend key proxy.
- **Data licensing** — Tatoeba sentences are CC BY 2.0 FR (attribution required), German Wiktionary content is CC BY-SA 4.0 + GFDL (share-alike — careful if you ever commercialise), Leipzig Wortschatz corpora are intended for academic use (verify per corpus), and **DW/Tagesschau audio is © ARD/Deutsche Welle** — link-out only, do not bundle.
- **The Goethe C1 spec changed in January 2024** to the modular format (230-word forum post + 120-word email in 75 min, with Reading at 65 min and Listening at 40 min). Older third-party guides still cite the legacy 80-minute / 200-word + 10-gap-letter format — design Module 1 to the new modular spec.
- **The Goethe "Deutsch für dich" community has been discontinued** — only the self-study exercises (200+) remain accessible. Do not promise Module 7 or Module 10 a "community" feature based on the older descriptions.
- **Claude-as-grader is not a Goethe examiner** — Module 10 should always disclaim that band estimates are indicative; for stakes-bearing prep, the user should still do at least one official Modellsatz under timed conditions.
- **Konjunktiv I has gaps** — when the K-I form coincides with the indicative (typical for plural and 1st-person), Konjunktiv II is the fallback. Module 3 must encode this rule, not just check against a single reference form.
